Webhook API Documentation

Overview

The webhook system for onemetric.in allows you to receive real-time notifications about specific events. This document outlines how to integrate and interact with the webhook system.

Endpoint Details

API URL https://your-webhook-url
Method POST
Content Type application/json
Authorization API key or Token in the headers

Request Contract

The payload for the webhook is a JSON object that adheres to the following structure:

{
  "event_type": "MARKET_GENERATED_ALERT",
  "sub_type": "Optional sub-type description",
  "time_stamp": "2024-10-01T10:20:00Z",
  "details": {
      "ltp": 30,
      "close": 20,
      "high": 35,
      "low": 15
  }
}

Each event_type represents a unique event. Examples include:

  • MARKET_GENERATED_ALERT
  • TRADE_EXECUTED
  • PRICE_THRESHOLD_REACHED
  • USER_NOTIFICATION_SENT
  • ...and more

Field Descriptions

Field Type Description
event_type string The type of event triggering the webhook.
sub_type string (Optional) A sub-category or additional context for the event.
time_stamp string The ISO 8601 timestamp when the event occurred.
details object An object containing event-specific data.

Example Response

The server should respond with a 200 HTTP status code upon successful receipt. For example:

{
  "status": "success",
  "message": "Webhook received successfully"
}

Notes

Important: Your webhook URL must be publicly accessible and use HTTPS for secure communication.
